A quick comment:

When we have dollar signs below like this, it just means that you type that line into the terminal (or CMD) and you don’t type the dollar sign. (Some people don’t know this so I thought it should be mentioned). The lines without dollar signs are simply the output like this:

Installing Node.js, npm, and Git

Windows

Sometimes we might talk about using Bash on Windows instead of CMD or PowerShell because it makes the development environments much better and fixes some issues when writing code that will eventually run on Linux.

It’s highly recommended to not use that here.

A lot of our Node.js development will involve using the Electron framework (which the text editor Atom is build from) and it does not get along with Bash on Windows / WSL for now. This is supposed to be resolved in the near-ish future.

You can definitely use Bash on Windows / WSL for other things, but it’ll raise problems for Node.js development.

Installing Node.js and npm

For macOS and Linux we’re using nvm (which is only available on those platforms). There is a similar alternative for Windows and you can install Node.js 6.11.0 this way if you’d like.

Otherwise install the LTS (6.11.0) from here. This should take care of adding Node.js and npm to your path.

After installing by either method, run this in CMD to update npm:

$ npm install -g npm@latest

Git

Download Git from here. This should simply work by calling git from either CMD or PowerShell. Otherwise, you may have to add some folders to your path.

The installation here is a lot different than on macOS and Linux. If you’re running into any issues I’d suggest looking them up online or maybe waiting until Tuesday in case anyone else was able to fix the problem.

I can’t help you much here since I’m more of a Mac / Linux guy, so sorry about that. :/

macOS

Xcode Command Line Tools

Before we do anything else please run this in your terminal (unless you have a version of macOS before Mavericks (OS X 10.9):

$ xcode-select --install

This will install some necessary development packages on your computer if you don’t already have them. If you want to know more about what this will add see here. If you have an old version of macOS then you might need to find other methods to get the command line tools.

Homebrew

Next we’re going to install the defacto package manager for the Mac, Homebrew. This line here will go ahead and install it for you.

$ /usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

# Let's put it on the path so we can go ahead and use it
$ export PATH=/usr/local/bin:/usr/local/sbin:$PATH

Installing Node.js and npm through nvm

So for macOS and Linux we’ll be using the Node Version Manager (or nvm) to install our version of Node.js. This will allow you to easily change your Node.js version on the fly if you wish.

# Installing nvm
$ curl -o- https://raw.githubusercontent.com/creationix/nvm/v0.33.2/install.sh | bash

# Now we'll go ahead and load it
$ export NVM_DIR="$HOME/.nvm"
$ [ -s "$NVM_DIR/nvm.sh" ] && . "$NVM_DIR/nvm.sh"

# With nvm we'll get Node.js and npm (this is the latest LTS for Node.js)
# Since you haven't installed anything else yet this will make it the default
$ nvm install 6.11.0

# We're also going to update the included version of npm
$ sudo npm install -g npm@latest

# To check that node and npm are working try the following:
$ node --version
v6.11.0
$ npm --version
5.0.4
# You should be getting v6.11.0 and an npm version >= 5 (at this time mine is 5.0.4)

Git

So you actually did install Git when getting the Xcode Command Line Tools, but we’re going to install with Homebrew as well so we can get bash-completion (this will make Git commands autocomplete and it’s pretty nice).

$ brew install git bash-completion

You’ll need to load in bash completion in your bash profile now, so add the following to the bottom of your ~/.bash_profile:

if [ -f $(brew --prefix)/etc/bash_completion ]; then
    . $(brew --prefix)/etc/bash_completion
fi

Linux

So for any Linux instructions we’re going to assume you have Debian, Ubuntu, or something similar, otherwise you might have to do things on your own.

Installing Node.js and npm through nvm

See the Installing Node.js and npm through nvm section for macOS to get Node.js and npm for Linux as well. The process is identical for Linux (although you don’t need to use sudo to update npm).

Git

To get Git with bash completion run this:

$ sudo apt install -y git bash-completion

and then put the following in your ~/.bash_profile:

if [ -f /etc/bash_completion ]; then
    . /etc/bash_completion
fi

GitHub Desktop

Unless you’re an amazing power user of Git, I would highly suggest getting GitHub Desktop. Unfortunately, this is not available on Linux, if this applies to you, you can probably look up other Git GUI programs if you so desire later on.

GitHub Desktop makes it easier to see what lines you are and aren’t committing which is why I tend to recommend it.

Atom

I highly suggest using Atom as it’s a pretty solid editor. Sublime is a good alternative if you prefer it.

After getting Atom we’ll add some packages onto it that’ll make things even easier.

Running this will add some stuff through the Atom package manager that’ll be handy. (If this isn’t working you can look up and add these packages separately through the Atom settings.)

$ apm install editorconfig es6-javascript atom-ternjs javascript-snippets linter linter-eslint language-babel autocomplete-modules

Go ahead and look around for some other cool Atom packages if you wish.
